## 1. 方法总览

PMcardio 采用**两阶段流水线**实现 ECG 图像的全自动数字化：

```
ECG 图像输入
     │
     ▼
┌─────────────────────────────┐
│  阶段一：ECG 归一化          │
│  (Normalization)            │
│                             │
│  Dotter网络 → 网格交叉点检测 │
│  Gridder算法 → 网格矩阵构建  │
│  Undistortion → 图像去畸变   │
└──────────────┬──────────────┘
               │ 标准化后的ECG图像
               ▼
┌─────────────────────────────┐
│  阶段二：ECG 重建            │
│  (Reconstruction)           │
│                             │
│  Leader网络 → 导联波形分割   │
│  后处理 → 信号精炼           │
│  格式自动检测                │
│  坐标转换 → 数字信号输出     │
└──────────────┬──────────────┘
               │
               ▼
       12导联数字ECG信号
```

**处理时间**：平均 4.86 秒（±0.6s），始终低于 10 秒。

## 2. 第一阶段：ECG 归一化 (Normalization)

归一化阶段的目标是：识别 ECG 图像上的所有主要网格交叉点，校正由纸张弯折、褶皱或透视变化引起的图像畸变，对坐标轴进行标定，生成标准化的网格结构。

### 2.1 Dotter 网络 - 网格交叉点检测

Dotter 是一个基于 U-Net + ResBlocks 的语义分割网络，专门用于检测 ECG 纸上的网格交叉点。

**网络架构与训练参数**：

| 参数 | 值 |
|------|-----|
| 基础架构 | U-Net + Residual Blocks (ResBlocks) |
| 输入尺寸 | 256 x 256 像素 patch |
| 输出 | 黑白分割掩码（白色=网格交叉点） |
| 训练轮数 | 300 epochs |
| 学习率 | 0.005 |
| 优化器 | ADAM |
| 损失函数 | Binary cross-entropy with logits |
| 激活函数 | SiLU (Sigmoid-weighted Linear Units) |

**训练数据**：

- 176 张 ECG 图像进行了交叉点标注
- 每张 ECG 平均标注 1,944（±649）个网格交叉点
- 训练集 140 张，验证集 36 张
- 通过数据增强生成超过 10,000 个 256x256 的输入 patch

**工作原理**：

1. 将 ECG 图像分割为 256x256 的 patch
2. 对每个 patch 进行推理，输出网格交叉点的分割掩码
3. 掩码中白色区域代表检测到的交叉点（dots）

> 注：此方法已申请专利（patent-pending）。

### 2.2 Gridder 算法 - 网格矩阵构建

Gridder 是 Dotter 输出的后处理算法，负责从离散的交叉点构建完整的网格矩阵：

1. 从 Dotter 检测到的交叉点出发
2. 处理因图像畸变导致的不规则分布
3. 通过插值填补缺失的网格交叉点
4. 生成最终的规整网格结构

### 2.3 Undistortion 算法 - 图像去畸变

利用检测到的网格交叉点对 ECG 图像进行去畸变处理：

**核心方法**：对每个网格方格单独应用**四点线性去畸变**（4-point linear undistortion）。

**处理流程**：

1. 输入：ECG 照片 + 精确的网格线交叉点坐标
2. 将图像按网格分割为独立的方格区域
3. 对每个方格应用四点透视变换，校正非线性纸张畸变
4. 将所有校正后的方格重新拼接
5. 输出：去畸变、标准化的 ECG 图像

此方法的优势在于能够处理纸张弯折、褶皱等造成的**局部非线性畸变**，因为每个网格方格独立校正。

## 3. 第二阶段：ECG 重建 (Reconstruction)

### 3.1 Leader 网络 - 导联波形分割

Leader 同样基于 U-Net + ResBlocks 架构，用于从归一化后的 ECG 图像中分割出导联波形。

**训练数据与方法**：

| 参数 | 值 |
|------|-----|
| 训练图像 | 232 张 ECG 图像 |
| 训练/验证划分 | 185 / 47 |
| 标注方法 | 逐层标注（每个可见导联一个独立的图层） |
| 验证方式 | 将标注层叠加到原始照片上进行目视验证 |
| 性能指标 | IoU（Intersection over Union / Jaccard 指数） |
| 增强数据 | 从训练集生成超过 10,000 个 patch |

**输出**：黑白掩码，检测到的导联用白色线条表示。

### 3.2 后处理流水线

Leader 网络的输出经过多步后处理精炼：

```
Leader 输出（导联掩码）
     │
     ▼
┌────────────────────────┐
│ 1. 噪声滤波             │  去除伪影和噪声
├────────────────────────┤
│ 2. ROI 识别             │  隔离相关信号区域
├────────────────────────┤
│ 3. 导联列分割           │  分离各导联的波形轨迹
├────────────────────────┤
│ 4. 基线识别             │  确定每个导联的基线
├────────────────────────┤
│ 5. 路径扩展             │  延伸已检测到的导联路径
├────────────────────────┤
│ 6. 复杂区域处理         │
│    - 连接导联端点        │
│    - 信号区域亮度最大化  │
│    - 重叠导联校正        │
│    - 信号间隙填补        │
└──────────┬─────────────┘
           │
           ▼
     精炼后的数字信号
```

### 3.3 格式自动检测

PMcardio 的一个关键创新是**自动 ECG 格式检测**，无需用户手动指定：

- **图像旋转方向**：自动检测正确的图像朝向
- **ECG 布局**：识别导联的具体排列方式
- **导联顺序**：判断标准顺序 vs Cabrera 排列
- **多页支持**：可处理跨多页的 ECG

格式描述为：`[行数] + [列数] + [节律导联]`

自动检测基于**肢体导联逻辑**和**节律导联信息**进行推断。

### 3.4 坐标转换

最后将像素坐标转换为实际 ECG 信号，需要两个用户输入参数：

- **纸速**（paper speed）：用于时间轴校准（通常 25mm/s）
- **电压增益**（voltage gain）：用于振幅校准（通常 10mm/mV）

## 4. 深度学习模型详情

### 架构：U-Net + ResBlocks

两个核心网络（Dotter 和 Leader）均采用相同的基础架构——在标准 U-Net 中集成了残差块（Residual Blocks）。

**U-Net 特点**：
- 编码器-解码器对称结构
- 跳跃连接保留空间信息
- 适合生物医学图像分割任务

**ResBlocks 增强**：
- 残差连接缓解梯度消失
- 允许更深的网络训练
- 提升特征提取能力

### Dotter 与 Leader 对比

| 特性 | Dotter | Leader |
|------|--------|--------|
| 任务 | 网格交叉点检测 | 导联波形分割 |
| 输入 | 256x256 patch | 归一化后的 ECG 图像 patch |
| 输出 | 交叉点二值掩码 | 导联波形二值掩码 |
| 训练数据量 | 176 张（140训练+36验证） | 232 张（185训练+47验证） |
| 增强 patch 数 | 10,000+ | 10,000+ |
| 评价指标 | — | IoU (Jaccard) |

## 5. 数据增强策略

### 训练时的数据增强

从训练图像生成超过 10,000 个增强 patch（256x256），涵盖以下变换：

#### 光照与颜色变换

| 增强方式 | 具体参数 |
|---------|---------|
| 亮度调整 | 模拟不同光照条件 |
| 色调变化 | 适应不同纸张和墨水颜色 |
| 对比度调整 | Gamma 校正：{0.125, 0.25, 0.5, 0.667, 1.5, 2, 4, 8} |
| HSV 亮度修改 | {±40, ±80, ±120, ±160} |

#### 几何变换

| 增强方式 | 具体参数 |
|---------|---------|
| 旋转 | {5, 15, 25, 35} 度 |
| 翻转 | 水平/垂直翻转 |
| 透视变换 | 模拟不同拍摄角度：{20°, 30°, 40°} |

#### 图像质量退化

| 增强方式 | 具体参数 |
|---------|---------|
| 模糊 | 核大小：{11, 21, 31, 41} 像素 |
| JPEG 压缩 | 质量因子：{90, 45, 22, 11, 5, 2} |
| 分辨率降低 | 缩减因子：{2, 4, 8, 16, 32} |

#### 信号层面增强

| 增强方式 | 具体参数 |
|---------|---------|
| 导联可见度修改 | 不透明度：{1, 0.6, 0.4, 0.2, 0.15} |
| 高频噪声（20-40Hz） | 低(0.02-0.05mV)，中(0.05-0.1mV)，高(0.1-0.15mV) |
| 低频噪声/基线漂移（0.05-0.3Hz） | 低(0.1-0.25mV)，中(0.25-0.5mV)，高(0.5-0.75mV) |

## 7. 性能指标与结果

### 整体性能（6,000 张图像）

| 指标 | 结果 |
|------|------|
| Pearson 相关系数 (PCC) | 平均 >91% |
| 均方根误差 (RMSE) | <0.10 mV |
| 信噪比 (SNR) | >12.5 dB |
| 平均处理时间 | 4.86 秒（±0.6s） |
| 时间范围 | 3.88 - 6.35 秒 |
| 失败率 | 6.62% |

### 按导联分析

- **表现最好**：I, II, III, aVR, aVL, aVF, V1, V2, V6
- **较有挑战**：V3-V5（信号振幅最大）
  - PCC 较低，RMSE 较高，SNR 较低
  - 原因：大振幅信号的数字化复杂度更高，且相邻导联容易重叠

### 按难度分析

| 指标 | 简单 (12子集) | 中等 (29子集) | 困难 (19子集) |
|------|-------------|-------------|-------------|
| PCC | 0.977 (0.975-0.981) | 0.970 (0.940-0.985) | 0.817 (0.087-0.968) |
| RMSE (mV) | 0.041 (0.037-0.043) | 0.047 (0.035-0.069) | 0.098 (0.048-0.254) |
| SNR (dB) | 16.682 (16.239-17.644) | 15.785 (11.922-21.225) | 11.101 (-0.759-22.058) |
| 处理时间 | ~4.5s (±0.15s) | — | 最高 6.35s |

### 高失败场景

失败率较高的场景包括：
- 极端缩放（Scale down 1/32）
- 严重模糊（Blur kernel 31, 41）
- 过度亮度降低（Brightness -160）
- 远距离拍摄（Zoom out extreme）

### 时间对齐方法（验证用）

为公平比较数字化信号与原始信号：
1. 允许 ±0.5 秒的时间偏移范围
2. 用 NaN 填充以找到最优偏移量
3. 通过减去中位差值进行垂直偏移校正

## 8. 与其他方法的对比

| 特性 | ECGScan | Imperial College London | ECGMiner | PMcardio（本方法） |
|------|---------|------------------------|----------|-------------------|
| 自动化程度 | 半自动 | 全自动 | 全自动 | **全自动** |
| 用户输入需求 | 大量 | 少量 | 少量 | 仅纸速+增益 |
| 自动格式检测 | 否 | 否 | 否 | **是** |
| 导联标签检测 | 良好 | 低分辨率下困难 | 存在问题 | **鲁棒** |
| 重叠信号处理 | 未说明 | 困难 | 有问题 | **鲁棒** |
| 依赖高质量图像 | 是 | 是 | 是 | **否** |
| 真实畸变处理 | 有限 | 有限 | 有限 | **全面** |
| 综合归一化 | 否 | 否 | 否 | **是** |
| 快速（<10s） | 否 | 是 | 是 | **是** |

### 具体对比

**vs ECGScan (Badilini et al., 2005)**：
- ECGScan 需要用户手动指定锚点，属于半自动方法
- PMcardio 完全自动化，无需锚点

**vs Imperial College London (Wu et al., 2022)**：
- 帝国理工方案在低分辨率图像的导联标签识别和重叠信号处理上表现不佳
- PMcardio 在这些场景下保持鲁棒性
- 帝国理工方案无自动格式检测

**vs ECGMiner (Santamonica et al., 2024)**：
- ECGMiner 在 PTB-XL 数据集上因重叠导联导致 14% 的拒绝率
- PMcardio 仅 6.62% 的失败率，且重叠处理更好
- ECGMiner 依赖高质量输入图像

## 9. 局限性与未来改进方向

### 当前局限

1. **起搏器脉冲处理**
   - 无法准确数字化起搏脉冲（pacing spikes）
   - 原因：持续时间极短、形态尖锐
   - 可能导致记录的 ECG 信号不准确

2. **极端图像退化**
   - 极端条件下仍有挑战
   - 需要基于图像质量评估的自动拒绝机制

3. **性能指标与临床相关性的不一致**
   - PCC, RMSE, SNR 并不总是与目视检查一致
   - 例：缺失 P 波可能显示良好的指标，但严重影响临床判读
   - 指标可能无法准确反映临床意义

4. **数据集覆盖范围**
   - PM-ECG-ID 可能无法涵盖更广泛临床实践中的全部 ECG 变异
   - 需在更多样化的患者群体中验证

### 未来改进方向

1. **自动分段评估**：将数字化 ECG 分割为 P 波、QRS 波群、T 波等临床相关组件，分别计算性能指标
2. **验证扩展**：在更多临床环境中验证，测试更多样化的患者群体
3. **算法改进**：解决极端图像退化场景，实现自动质量评估与拒绝，开发起搏脉冲检测功能
