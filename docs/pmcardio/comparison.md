# Open-ECG-Digitizer 与 PMcardio 后处理方案对比分析

> 本文档基于我们的框架文档 `docs/framework/07-post-processing.md` 与 PMcardio 论文的后处理描述进行对比。
>
> PMcardio 论文对后处理仅有三句话的描述，很多细节未公开（可能因专利保护）。对比中标注了 `[PMcardio 未公开]` 的部分为论文未提供具体实现的环节。

---

## 1. 架构层面的根本差异

| 维度 | Open-ECG-Digitizer | PMcardio |
|------|---------------------|----------|
| **网络输出** | 4 通道 softmax 概率图（signal / grid / text / background） | 二值 mask（白线 = 导联） |
| **信息量** | 概率值连续，可做加权质心；grid 和 text 独立通道 | 仅有前景/背景，无语义区分 |
| **去畸变时机** | 后处理中完成（阶段 2-5） | 第一阶段（Normalization）已独立完成 |
| **导联识别** | 专用文本检测 U-Net + 布局模板匹配 | 启发式后处理分类 `[PMcardio 未公开]` |
| **整体架构** | 单阶段后处理流水线 | 两阶段分离（归一化 + 重建） |

PMcardio 将去畸变和信号提取分成两个独立阶段（先归一化再重建），我们在一个后处理流水线中连续完成。

---

## 2. 逐步骤对比

### 2.1 噪声过滤

| | Open-ECG-Digitizer | PMcardio |
|---|---|---|
| **方法** | `process_sparse_prob`：减均值 → clamp(0) → 归一化到 [0,1] | "filtering noise" `[PMcardio 未公开]` |
| **操作对象** | 概率图（连续值） | 二值 mask（推测为连通域过滤） |
| **优势** | 概率图保留渐变信息，后续加权质心可利用 | — |

**差异**：我们在概率空间做降噪，PMcardio 在二值空间做降噪。概率空间的优势是保留了"半确定"区域的信息，不会因二值化硬阈值丢失边缘细节。

---

### 2.2 ROI 识别与透视校正

| | Open-ECG-Digitizer | PMcardio |
|---|---|---|
| **ROI 方法** | Hough 变换检测网格线角度 → 百分位裁切 → 透视变换 → Y 方向裁切 | "identifying ROI" `[PMcardio 未公开]` |
| **透视检测** | 两遍 Hough（粗搜 + 精搜）、Hann 窗加权、方差矩阵选角 | 无独立步骤（归一化阶段已处理） |
| **去畸变** | 全局透视变换 + 可选 TPS 去畸变（球谐卷积 + KNN 图 + 梯度优化，默认关闭） | 第一阶段 Undistortion：逐网格方格四点线性去畸变 |

**差异**：
- 我们的透视检测和 ROI 裁切有完整算法实现，PMcardio 在此阶段仅提到"identifying ROI"。
- PMcardio 的去畸变在归一化阶段更精细——对每个网格方格独立做四点线性校正，可处理局部非线性畸变（纸张褶皱、弯折）。我们的全局透视变换无法处理局部畸变，TPS 去畸变可以但默认关闭。

---

### 2.3 网格间距估计

| | Open-ECG-Digitizer | PMcardio |
|---|---|---|
| **方法** | 自相关 + 多级缩放网格搜索（10 轮 zoom，每轮 1000 候选） | 无独立步骤 |

**差异**：这是我们独有的步骤。PMcardio 在归一化阶段通过 Dotter 网络检测网格交叉点时就隐含了间距信息，不需要再单独估计。我们没有显式的网格交叉点检测网络，所以需要自相关方法从 grid_prob 中估计 mm/pixel。

---

### 2.4 信号提取（差异最大的环节）

| | Open-ECG-Digitizer | PMcardio |
|---|---|---|
| **分割策略** | 连通组件标记 → 分 4 条带重标记 → 面积过滤 → 概率加权质心提取 | 按 lead columns 切分 `[PMcardio 未公开]` |
| **轨迹表示** | 概率加权质心：`y[col] = Σ(y × prob) / Σ(prob)`，天然亚像素精度 | 二值 mask 上的路径追踪 |
| **断裂处理** | 匈牙利算法全局最优线段合并（代价 = 距离 × 高度差惩罚，支持环绕） | path extension + endpoint connection `[PMcardio 未公开]` |
| **重叠处理** | 迭代提取（最多 4 轮）+ 拒绝区域水平路径追踪分离 | maximizes brightness + correct overlapping leads `[PMcardio 未公开]` |
| **行数检测** | 自动统计"零→非零"过渡次数，取众数 | 自动格式检测（旋转 / 布局 / 导联顺序） |

**关键差异**：

1. **加权质心 vs 路径追踪**：我们用概率加权质心提取轨迹，天然具备亚像素精度，且概率信息作为权重本身就在做 PMcardio 所说的"maximizes brightness"——概率高的像素贡献更大。PMcardio 在二值 mask 上做路径追踪，需要额外的"maximizes brightness"步骤来选择最优路径。

2. **匈牙利算法 vs 启发式连接**：我们用匈牙利算法做全局最优线段匹配，代价函数综合了水平距离、垂直距离和高度差惩罚（30 倍权重），还支持 rhythm strip 的环绕合并。PMcardio 用局部启发式的端点连接。全局优化理论上更不容易"接错线"。

3. **迭代提取 vs 单次处理**：我们最多做 4 轮迭代提取，每轮对拒绝区域做水平路径追踪分离重叠信号，再更新概率图重新提取。这对应 PMcardio 的重叠导联修正，但策略不同。

---

### 2.5 基线估计

| | Open-ECG-Digitizer | PMcardio |
|---|---|---|
| **方法** | `lines -= lines.nanmean(dim=1)`（逐行去均值） | "identifies the baseline for each lead" `[PMcardio 未公开]` |
| **时机** | 信号提取完成后，在归一化阶段统一处理 | 导联列分割后、路径延拓前 |

**差异**：我们用均值去基线，PMcardio 的方法未知。均值方法对 QRS 大幅偏转的鲁棒性可能稍弱（均值被尖峰拉偏），但我们是对已提取的连续轨迹操作，而非对原始 mask 操作，影响相对可控。PMcardio 将基线估计放在信号提取流程中间（分割导联列之后、路径延拓之前），而我们放在提取完成之后。

---

### 2.6 导联识别与归一化

| | Open-ECG-Digitizer | PMcardio |
|---|---|---|
| **导联标签检测** | 专用文本检测 U-Net（1→13 通道） + 阈值 0.8 过滤 | 自动格式检测 `[PMcardio 未公开]` |
| **布局匹配** | 遍历模板库，最小二乘估计缩放+平移，残差+缺失惩罚选最优 | 基于肢体导联逻辑和节律导联信息 `[PMcardio 未公开]` |
| **翻转检测** | 可选，遍历时同时检查翻转版本 | 自动检测图像旋转方向 |
| **Cabrera 检测** | 余弦相似度 > 0.992 时限制搜索为 Cabrera 布局 | 支持 Cabrera，方法未公开 |
| **Rhythm 导联匹配** | 余弦相似度 + 匈牙利算法，常见 rhythm 导联加权 | — |
| **单位转换** | `scale = (0.1 mV/mm / avg_pixel_per_mm) × 1000` → µV/pixel | 用户输入纸速 + 电压增益 |

**差异**：我们有独立的文本检测网络和系统化的布局模板匹配，PMcardio 用启发式方法但支持自动旋转检测。我们的单位转换依赖自动估计的 mm/pixel，PMcardio 需要用户输入纸速和增益。

## 3. 我们有但 PMcardio 未提及的能力

| 能力 | 我们的实现 |
|------|-----------|
| **概率图加权质心** | 亚像素精度信号提取，概率值天然作为置信度权重 |
| **匈牙利算法线段合并** | 全局最优匹配，代价函数含距离+高度差惩罚，支持 rhythm strip 环绕 |
| **TPS 去畸变** | 球谐卷积检测交点 + KNN 图 + 梯度优化 1000 步 + TPS 拟合（默认关闭） |
| **专用文本检测 U-Net** | 独立的 13 通道导联标签识别网络 |
| **布局模板库** | `lead_layouts_all.yml` 中定义的系统化多布局匹配 |
| **自动行数检测** | 基于过渡次数统计，取众数 |
| **迭代提取** | 最多 4 轮，拒绝区域水平路径追踪分离重叠信号 |
| **自动 mm/pixel 估计** | 无需用户输入纸速/增益 |

---

## 4. PMcardio 有但我们缺失或较弱的能力

| 能力 | PMcardio 做法 | 我们的现状 | 改进建议 |
|------|--------------|-----------|---------|
| **逐网格方格去畸变** | Dotter 网络检测交叉点 → Gridder 建网格 → 逐方格四点线性校正 | 全局透视变换（局部畸变处理较弱），TPS 默认关闭 | 考虑启用 TPS 或实现类似的逐方格校正 |
| **显式路径延拓** | 沿轨迹方向外推延伸断裂端点 | 无显式外推，依赖匈牙利算法跨 gap 合并 | 可在匈牙利匹配前加入端点方向外推 |
| **多候选路径选择** | "maximizes brightness"——在歧义区域选响应最强的路径 | 加权质心隐含了此能力，但无显式多候选机制 | 在重叠区域可增加候选路径评分机制 |
| **自动旋转检测** | 自动判断图像正确朝向 | 仅判断 H > W 时旋转 90° | 可增加更智能的旋转检测逻辑 |
| **多页 ECG 支持** | 明确支持跨多页 ECG | 未提及 | 按需添加 |

---

## 5. 核心结论

1. **信号提取策略本质不同**：我们走"概率加权质心"路线，PMcardio 走"二值 mask 路径追踪"路线。前者信息利用率更高、有亚像素精度；后者在极端断裂/重叠情况下可能更灵活（可做方向外推和多路径评分）。

2. **断裂修复策略不同**：我们用匈牙利全局最优匹配，PMcardio 用局部启发式连接。全局优化在一般情况下更优，但局部启发式在高密度重叠区域可能更灵活。

3. **去畸变精度差异**：PMcardio 的逐网格方格独立校正理论上比我们的全局透视变换更能处理局部畸变（纸张褶皱、弯折）。我们的 TPS 去畸变可以弥补但默认关闭，可考虑评估启用效果。

4. **重叠导联处理**都是难点，两者都没有完美解决。我们的迭代提取 + 水平路径追踪是一种方案，PMcardio 的多约束联合决策是另一种思路。V3-V5 高振幅导联在两种方案下都是最大挑战。

5. **自动化程度**：PMcardio 在旋转检测和格式检测上更自动化；我们在网格间距估计和单位转换上更自动化（不需要用户输入纸速/增益）。
